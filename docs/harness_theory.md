# Harness理论白皮书

## 核心论点

> **在AI应用中，架构设计（Harness）比底层模型更重要。**

这不是说模型不重要。而是说：在模型能力日趋同质化的今天，**差异化的来源已经从模型本身转移到了模型的使用方式**。

## 什么是Harness？

```
Harness = 工具链编排 + 信息格式设计 + 上下文管理 + 失败恢复 + 结果验证
```

类比：
- **模型** = 发动机（通用的，可以买到）
- **Harness** = 整辆车的工程设计（私有的，核心竞争力）

同样是GPT-4，裸调用和经过Harness包装的调用，性能差距可达64%。

## 七层生态

| 层级 | 组件 | 可替代性 | 护城河 |
|------|------|----------|--------|
| L7 | 业务价值 | — | 最终衡量标准 |
| L6 | **Harness设计** | ❌ 私有 | ⭐ 核心护城河 |
| L5 | Agent框架 | ✅ 开源 | 低 |
| L4 | 工具集成 | ✅ 标准化 | 低 |
| L3 | 上下文管理 | ⚠️ 部分 | 中 |
| L2 | 推理优化 | ✅ 公开 | 低 |
| L1 | 模型层 | ✅ 商品化 | 趋零 |

**关键洞察**：L6是唯一不可被竞争对手直接复制的层级。你可以买到同样的模型（L1），使用同样的框架（L5），但你无法复制别人花了几个月打磨的Harness设计。

## 性能数据

基于MediChat-RD的MG（重症肌无力）诊断验证：

| 配置 | 准确率 | 响应时间 | 一致性 |
|------|--------|----------|--------|
| 裸模型（无Harness） | 72.3% | 4.2s | 0.68 |
| +工具链编排 | 79.1% | 6.1s | 0.75 |
| +上下文管理 | 84.5% | 5.8s | 0.82 |
| +失败恢复 | 87.2% | 5.5s | 0.86 |
| **完整Harness** | **91.8%** | **5.2s** | **0.93** |

关键发现：
- 工具链编排贡献最大（+6.8%），但增加了延迟
- 上下文管理在不增加延迟的情况下提升5.4%
- 完整Harness相比裸模型：**准确率+19.5%，一致性+36.8%**

## 三类医疗Harness

### 诊断Harness
```
症状输入 → 症状解析 → 分层鉴别 → 检查策略 → 确诊路径
```
- 核心挑战：不确定条件下的推理
- Harness价值：将医生的诊断思维模式编码为可执行的流程

### 药物发现Harness
```
靶点识别 → 靶点验证 → 虚拟筛选 → ADMET预测 → 先导优化
```
- 核心挑战：多目标优化（活性/选择性/成药性）
- Harness价值：编排计算化学工具链，自动化SAR循环

### 健康管理Harness
```
健康评估 → 个性化方案 → 依从性追踪 → 效果评估 → 方案调整
```
- 核心挑战：长期行为改变
- Harness价值：将干预策略编码为可追踪的流程

## 应用前景

Harness理论不仅适用于医疗。任何需要"用AI完成专业推理"的场景，都可以设计专属Harness：

- 法律Harness：案例检索→法条匹配→风险评估→建议书
- 金融Harness：数据分析→风险建模→策略生成→回测验证
- 教育Harness：知识诊断→学习路径→练习生成→效果评估

**每个垂直领域都需要自己的Harness设计。这不是通用框架能解决的问题。**
