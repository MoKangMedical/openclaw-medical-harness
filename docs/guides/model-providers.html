Model Providers Guide

OpenClaw-Medical-Harness supports pluggable model providers. The model is intentionally the most replaceable component — your competitive advantage lives in the harness design.

MIMO (Default)

from openclaw_medical_harness import MIMOProvider

# Option 1: Pass key directly
provider = MIMOProvider(api_key="sk-xxx")

# Option 2: Environment variable
# export MIMO_API_KEY="sk-xxx"
provider = MIMOProvider()

# Use with harness
harness = DiagnosisHarness(specialty="neurology", provider=provider)
MIMO API Details
Base URL: https://api.xiaomimimo.com/v1
Models: mimo-v2-pro (default), mimo-v2-flash
Format: OpenAI Chat Completions compatible

Custom Provider

from openclaw_medical_harness.providers.base import ModelProvider, GenerationResult

class MyProvider(ModelProvider):
    @property
    def name(self) -> str:
        return "my-provider"

    @property
    def model(self) -> str:
        return "my-model"

    def generate(self, prompt: str, **kwargs) -> GenerationResult:
        # Your API call here
        return GenerationResult(text="response", model=self.model)

# Register and use
from openclaw_medical_harness.providers import register_provider
register_provider("my-provider", MyProvider)
provider = create_provider("my-provider")

Fallback Behavior

If no model provider is configured (no API key), the harness operates in fallback mode using knowledge base matching only. This is useful for testing but not recommended for production.
