Three-Layer Harness Architecture

🩺

Diagnosis Harness

Structured diagnostic reasoning: symptoms → differential diagnosis → workup strategy. Supports rare disease knowledge base, OMIM lookup, and PubMed evidence retrieval. Validated with Myasthenia Gravis and Spinocerebellar Ataxia cases.

💊

Drug Discovery Harness

Drug discovery pipeline: target validation → virtual screening → ADMET prediction → lead optimization. Integrates ChEMBL for compound data, OpenTargets for target-disease associations. Returns real candidate compounds with IC50 values.

🏃

Health Management Harness

Personalized health management: assessment → intervention plan → adherence tracking → effectiveness evaluation. Supports multi-modal data (labs, wearables, questionnaires). Generates actionable diet and exercise prescriptions.

🤖

Pluggable Model Providers

Supports Xiaomi MIMO (default), Claude, GPT-4, and local Ollama. The model layer is intentionally replaceable — your competitive advantage lives in the harness design, not the model.

🔧

MCP Tool Integration

Pre-configured tools: PubMed (literature), ChEMBL (compounds), OpenTargets (targets), OMIM (genetics), OpenFDA (safety). Real API calls with rate limiting and error handling.

🛡️

Medical Safety Validation

Multi-dimensional validation: format, consistency, domain rules, safety checks. Detects absolute claims ("definitely cancer"), dangerous patterns, and missing differential diagnoses for high-risk conditions.

Quick Start

from openclaw_medical_harness import DiagnosisHarness, MIMOProvider

# Create harness with MIMO model
provider = MIMOProvider(api_key="your-key")
harness = DiagnosisHarness(specialty="neurology", provider=provider)

# Run diagnosis
result = harness.execute({
    "symptoms": ["bilateral ptosis", "fatigable weakness", "diplopia"],
    "patient": {"age": 35, "sex": "F"}
})

print(result["diagnosis"])    # Myasthenia Gravis
print(result["confidence"])   # 0.85
print(result["differential"]) # ["Lambert-Eaton", "Botulism", "CPEO"]
print(result["next_steps"])   # ["AChR antibody", "RNS", "Ice pack test"]

Frequently Asked Questions

What is OpenClaw-Medical-Harness?

OpenClaw-Medical-Harness is an open-source medical AI agent orchestration framework built on Harness Theory. It provides a structured pipeline (Tool Chain → Context Management → Model Reasoning → Validation → Recovery) that makes any LLM perform medical reasoning reliably. Version 0.2.0 integrates with Xiaomi MIMO, Claude, and GPT-4 models.

What is Harness Theory in AI?

Harness Theory proposes that in AI applications, the system architecture (harness) matters more than the underlying model. A well-designed harness can improve performance by up to 64%. The harness includes tool chain orchestration, information format design, context management, failure recovery, and result validation. Unlike the model (which is replaceable), the harness is the private competitive advantage.

How does the Medical Harness handle drug discovery?

The Drug Discovery Harness follows a 4-stage pipeline: (1) Target validation using OpenTargets API, (2) Virtual screening with ChEMBL compound database, (3) ADMET prediction for absorption, distribution, metabolism, excretion, and toxicity, (4) Lead optimization suggestions. It integrates with real APIs including PubMed, ChEMBL, and OpenTargets.

What model providers are supported?

The framework supports pluggable model providers: Xiaomi MIMO (mimo-v2-pro, default), OpenAI GPT-4, Anthropic Claude, and local Ollama models. You can also implement custom providers by extending the ModelProvider base class.

Is it suitable for rare disease diagnosis?

Yes. The framework includes a rare disease knowledge base and integrates with OMIM for genetic disease lookup. The Diagnosis Harness performs differential diagnosis with confidence scoring and has been validated with rare neurological conditions including Myasthenia Gravis and Spinocerebellar Ataxia.
